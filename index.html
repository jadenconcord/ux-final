Task Management - UX Project



For many Task Management is a challenging task. Though the concept of Task Management is quite simple, the effectiveness of keeping track of tasks is sometimes ineffective through various software methods. Through analysis, research and experiments of Task Management I have found the optimal software solution for Task Management.

Three aspects of the optimal task management application are researched. Firstly, the flow of managing tasks. This includes the framework for how tasks are displayed, calculated and handled. Secondly, the layout of the application including the header, sidebar and content position and necessity. Lastly, the navigation of the site's header links.

Why a New Task Management System?

Personally, I have never used task management system that has been effective enough for me to fully implement. I have noticed may flaws in current task management software and written methods where I have found the optimal method that should be used. To stand out from competitors, the new system focuses on simplicity while still having the functional features to be successful.

Purpose

The project I am focusing on could effect how I keep track of and manage tasks. I am invested in this project because I want to find the best method for task management. I currently do not have the best task management system, so this project allows me to create or use a better task management system.

Personas

The following personas have been made out of mainly interviews and experiments on three different groups. Not real people.

Features / Goals

  • View Tasks
  • View and set dew dates
  • Create subject/class topics for student
  • Calender view or a similar way to display events
  • Tasks within tasks
  • [student] Allow work, school, and other to be separated from each other
  • [manager] Manage projects within categories
  • [employee, manager] Share tasks (and groups) with other people
  • [employee] Hierarchy of tasks
  • [manager] Multiple reminders per event/task
  • [student] Task reminders in advance with custom times
  • [manager] Recurring events/tasks

Joe - Student

Joe is a high school student who is involved in various clubs and extracurricular activities. Joe struggles to keep track of tasks and events for his school and activities. He is good at remembering projects and tasks he needs to get done, but because of this, he has problems turning in things on time and not forgetting things.

Sally - Manager

Sally is a experienced manager in a corporate environment. She has moved up in her position through her hard work and motivation. She is very productive and makes sure to keep track of things, however she is not the best at working efficiently. She also finds using technology challenging, preferring to use written methods to keep track of things, however this comes as a disadvantage to her at times.

Matt - Software Engineer

Matt is a junior software Engineer at a corporate software company. Matt is good at his job, but finds focusing on his work challenging. He easily gets destructed and easily forgets events and projects that need to be done. He is very busy with projects and often has very little time to do other things besides work. He uses task management but finds that he forgets reminders after he sees them sometimes.

User Stories

  • Student
    • As a Student, I want to separate my work and school tasks from each other so that I can stay more organized and focus on specific groups at a time
    • As a Student, I want to be able to be reminded of events in advance so that I can prepare for the next event
    • As a Student, create subjects for my classes so that I can focus on a single class at a time and stay organized
  • Manager
    • As a Manager, I want to set recurring events so that I don't have to create a new event every time
    • As a Manager, I want to view my co-workers tasks so that I can see what they have completed
    • As a Manager, I need to be able to create projects that I can share with my co-workers so that I can easily give then their work
  • Employee
    • As an Employee, I need to show my tasks to my manager so that I can get credit for the work I have done
    • As an Employee, I need to prioritize my tasks so that I can be able to remember that more important tasks that I am told to complete first
    • As an Employee, I need to create large projects with sub-tasks so that I can stay organized withing my larger scope of tasks

Wireframes

The first frame was made out of the results from the layout experiments which showed me the following information

  • Using the sidebar is the most efficient and easy to learn layout
  • Using a dropdown box is the most challenging item for certain users to understand
  • Dropdown boxes should be expanded by default to prevent confusion in not understanding opening them

For the header, I also had the card sort which revealed that I should combine the settings and account together.

In the next iteration, I did research on task management, and integrated its functionality into the wireframes (shown to the right)

From looking at the card sort, I was also able to use 4 items on the menu which are Upgrade, Preferences, Account Settings, and help. The upgrade section was added as the call to action.

In addition to these, I added catagories because they were needed from the personas and needs of general users.

The next iteration added a basic theme to the app which made it easier to understand the functionality of the app. I also filled in placeholder text and added the sharing options, again because of the needs shown in the personas and research.

The last iteration outlines and identifies the functionality of the app. Note: the logo icon on the top left is a placeholder for the client's or app's logo.

The last feature I needed to add was to tasks. I added options to create sub-tasks. This action was done to give advanced users options to have more freedom on how they organize the app. Basic functionality is simple, but this is an advanced feature.

I also have added an assigned icon to show that a task is assigned or not. It is also toggle-able so the the tasks do not need to be edited in an external popup. The task names are also editable when focused.

Conclusion

A concluding summary of your project that addresses how your designs improve the user experience beyond the current state and/or create business value.

At the beginning of this project, I analyzed existing task management apps and then stripped the basic feature from them, creating a more simplified, user oriented, effective design that gives a competitive edge over the competitors.

The best designers and the best programmers aren't the ones with the best skills, or the nimblest fingers, or the new who can rock and roll with Photoshop or their environment of choice, they are the ones that can determine what just doesn't matter. That's where the real gains are made.
     - 37signals

By recognizing that simplicity is a competitive advantage, I was able to conduct research and experiments that allowed me to create a more efficient task management solution. These experiments allowed me to find the optimal layout for the app, which turned out to include a sidebar. Using research into task management, I was able to add features to improve productivity in the app more than its competitors.

With the competitive advantage, I was able to surface, higher conversion rates and a greater use of the app is to be expected through the improvements to usability. With the addition of premium features, the application should generate high business value with a more pleasing application.
